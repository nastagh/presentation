SVG

Scalable Vector Graphics (SVG) are an XML-based markup language for describing two-dimensional based vector graphics.

Advantages Disadvantages
  • small size
  • flexibility
  • scalability without changing the image quality
  • it’s possible to view the contents of the SVG file in any browser
  • file size
  • read only the entire object

    • How to use SVG images

    SVG as an <img>

    <img src = "happy.svg" alt="My Happy SVG"/> Advantages:
  • SVG animations and filters are supported
    • Disadvantages:
  • you can't change the properties of SVG elements via CSS or JS
  • interactive animations are not supported

    • SVG as an CSS background-image

    <body> {

    background-image: url(happy.svg)

    }

    </body>

    • SVG as an <object>

    <object type="image/svg+xml" data="happy.svg" width="300" height="300" class=”icon”> </object> Advantages:
  • you can use an external CSS file to manage styles
  • SVG animations and filters are supported
  • interactive animations are supported

    • SVG as an <iframe>

    <body>

    <iframe src="happy.svg" width="400px" height="400px"></iframe>

    </body>
